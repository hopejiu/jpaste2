## 工具箱新增两模块：二维码生成 + SVG 转 PNG（grill-me 收敛后实现）
关键决策：**新增第三方依赖 = 0**。
- **二维码 = 后端 rxing**（已是依赖，仅用于解码，这次加编码调用）。新增 `command/image_export.rs::generate_qr(content,size,ec_level,margin,fg,bg)->base64 PNG`，用 `QRCodeWriter.encode_with_hints` + `EncodeHints{ErrorCorrection,Margin,CharacterSet}`，遍历 BitMatrix 用 `image::RgbaImage` 上色（fg/bg hex 解析），`base64` crate 编码（原为传递依赖，加为直接依赖零成本）。前端 200ms 防抖实时预览。
- **SVG→PNG = 纯前端 canvas**（`svg-view.tsx`）。两层污染检测：①`detectExternal()` 预扫源码 http(s)/url()/@font-face/foreignObject → 直接报错；②`canvas.toDataURL` try/catch 捕获 SecurityError。输入源：文本框 + `<input type=file accept=.svg>`(FileReader) + 读剪贴板。缩放 1x/2x/3x（乘 naturalWidth/Height）。
- **导出双按钮（复制+保存）** 两模块共用：`write_clipboard_image(bytes)`（clipboard-rs `RustImageData::from_bytes`+`set_image`，需 `use RustImage` trait）、`save_image_dialog(bytes,name)`（tauri-plugin-dialog `blocking_save_file`+fs::write）。
- **生成图不回写历史**：`SelfWriteTracker` 加 `mark_image()`/`is_image_self_write()`（时间窗抑制 IMAGE_TTL=1500ms，因 PNG 重编码 round-trip 哈希不匹配，故用计时器非内容哈希），watcher `on_clipboard_change` 检测 has_image 时若命中则跳过。
- 另：`get_clipboard_text()` 供 SVG 读剪贴板输入。窗口尺寸 QR 520×620 / SVG 640×640。viewer.rs VIEWERS 加两行，toolbox TOOLS 加两项（图标 qrCode/image 均已存在）。
- 注意：`cargo test` 跑 lib 测试报 STATUS_ENTRYPOINT_NOT_FOUND(0xc0000139)——WebView2 DLL loader 环境问题，**所有测试**（含既有 qrcode 测试）都这样，非本次代码问题；`cargo check --lib` + `tsc --noEmit` 均通过。

## 排查 exe 偶尔自动退出：加 panic 捕获日志
根因方向：release 配置 `panic = "abort"`，任何线程 panic 直接终止整个进程且不经过日志系统，故"偶尔自动退出"后日志无迹可寻。
改动：
- `log_service.rs`：新增 `APP_DATA_DIR` OnceCell（init 时填充）+ `crash_log(line)` 独立开文件句柄追加写当前日志文件（绝不锁共享 writer Mutex，避免 panic 时 Mutex 中毒/被持有时死锁），同时写 stderr；logger 未初始化时回退到 temp 目录 `jpaste-crash-HH.log`。
- `lib.rs::run()` 开头 `std::panic::set_hook`：捕获 payload（&str/String）、location、thread name、`Backtrace::force_capture()`，格式 `[CRASH] ...` 调 `crash_log`。
- `lib.rs` setup 中 `build_services` 后新增 `log::info!("Log file: {}", app_data.join("logs").display());` 告知日志路径。
- `Cargo.toml` release 改 `strip = false` + `debug = 1`（保留符号，panic backtrace 才能解析出函数名/行号；定位后改回 `strip = true`）。
日志位置：app_data_dir/logs/jpaste-HH.log（Windows 一般 `C:\Users\<user>\AppData\Roaming\com.jpaste.app\jpastes\logs`）。验证：`cargo check` 通过。
