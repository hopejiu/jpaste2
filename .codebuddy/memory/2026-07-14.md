## jPaste2 图片项点击/Enter 行为修正（第二版）
- 根因1：图片条目 Enter 只复制不粘贴 —— `paste_entry` / `paste_entry_and_hide` 原只 `write_text`，无图片分支；`handleSelect` 对图片无条件走 copyEntry。修复：`command/clipboard.rs` 抽出 `write_entry_to_clipboard(state,id)`（有 image_path 则 read+write_image 返回 true，否则 write_text），copy_entry/paste_entry/paste_entry_and_hide 共用；`use-main-page.ts` 图片分支尊重 `default_action`，paste 时走 `pasteEntryAndHide`（写图片+模拟粘贴+隐藏）。
- 根因2：点击图片非缩略图部分仍放大 —— `entry-item.tsx` 的查看窗口 onClick 绑在 `entry-image-preview` 包裹 div 上，div 内图片周围空隙点击也触发放大。修复：onClick 移到 `<img>` 本身，只有实际图片像素打开查看窗口，包裹 div / 元数据区点击穿透到 onActivate（复制/粘贴）。
- cargo check + tsc 通过。

## jPaste2 图片项点击 / Enter 行为修正
- 根因：图片条目的 `entry.content` 为空（图片以文件存于 `image_path`），原 `handleSelect` 走 `copyToClipboard("")` 静默无操作 → 只有缩略图（onImageClick）能打开查看窗口，正文区点击和 Enter 都「没反应」。
- 后端 `command/clipboard.rs` 新增 `copy_entry(id)`：有 `image_path` 则读文件 `write_image` 复制真实图片，否则 `write_text`。`lib.rs` 注册。
- 前端 `invoke.ts` 加 `copyEntry`；`use-main-page.ts` 的 `handleSelect` 对 `entry.has_image` 走 `copyEntry`（含计数+自动隐藏）。缩略图仍走 `onImageClick` 打开查看窗口（entry-item 未改）。
- 结果：图片项只有缩略图点开新窗口，其余区域/Enter = 复制图片（复制/粘贴）。cargo check + tsc 通过。
