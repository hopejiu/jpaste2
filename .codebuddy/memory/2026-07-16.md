# 2026-07-16

## 修复：快速启动网页窗口最小化后用快捷键无法呼出
- 根因：quicklaunch.rs `launch_web` 在窗口已存在且 `is_minimized()` 为 true 时，只调用 `show_focus_window`（仅 `show()` + `set_focus()`）。Windows 上 `show()` 不会恢复最小化窗口，导致窗口停在任务栏无法呼出。
- 修复：在 minimized 分支先 `window.unminimize()` 再 `show_focus_window`。改动仅限 src-tauri/src/command/quicklaunch.rs。
- 注意：cargo 因 D: 盘 reparse point 被 PowerShell 视为不受信任挂载点而无法运行，未能本地编译验证；但 `unminimize()` 是 Tauri v2 `WebviewWindow`（deref 到 `Window`）的合法方法。

## 修复：share-panel 拖拽报错 `plugin:event|listen not allowed by ACL`
- 根因：`capabilities/default.json` 的 windows 只为 `["main","toast-0","*-viewer-*"]` 授权，share 面板窗口 label 是 `share-panel`，不匹配任何模式，因此拿不到 `core:event:allow-listen`，`onDragDropEvent` 监听被 ACL 拒绝。
- 修复：在 windows 数组加入 `"share-panel"`。改动限 src-tauri/capabilities/default.json。

## 修复：HTTP 共享下载由纯 JS 改为直链
- 背景：下载原用 `downloadFile()`（fetch+blob+anchor 触发），分别在 Rust `render_list_page` 服务端初始渲染和 `client.js` 的 `renderItem` 前端刷新渲染两处。
- 修复：两处下载按钮改为 `<a class="btn" href="/d/{id}">`，依赖 download handler 已返回的 `Content-Disposition: attachment` 触发原生下载；删除 `client.js` 中已变死代码的 `downloadFile()`；给 `.btn` 加 `text-decoration:none`（anchor 默认下划线）。改动限 share_server.rs / share-client/client.js / share-client/styles.css。
- 注：原注释称 Android Chrome 忽略原生 `<a download>`，但普通 `<a>`（无 download 属性，靠 Content-Disposition）在移动端可靠触发下载。
