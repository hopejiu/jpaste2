# 2026-07-22

## 修复 jpaste2 全部排序选项不生效
根因：`use-entries.ts` 的 `refreshEntries()`/`loadMore()` 调用 `api.getEntries` 时未传 `sortField`/`sortOrder`，后端始终用默认 `updated_at`/`DESC`。所以所有排序（最新/最旧/最多使用/最长/最短）都不生效，不只是"最短"。
修复：
- 排序 signal 从 `use-main-page.ts` 移到 `use-entries.ts`（查询逻辑所在处），前端两函数与 regex 路径都传入 sort 参数。
- `invoke.ts` 的 `getEntriesRegex` 接受 sortField/sortOrder。
- 后端 `entries.rs` 游标字段从 `copy_count ?: updated_at` 改为按 `sort_field` 本身（content_length 排序分页才一致）。
改为后 `pnpm exec tsc --noEmit` 与 `cargo check` 均通过。

## 列表页收藏 icon 位置一致 + action 图标去重
需求1：收藏项常驻星标位置须与未收藏 hover 星标一致（之前常驻星标因与 `.entry-actions` 竞争 `margin-left:auto` 偏左）。
改动 `entry-item.tsx`：将 fav 按钮改为 `.entry-actions` 的【最后一个子元素】（far-right），`.entry-actions` 去掉默认 `margin-left:auto`，改用条件类 `push-right`(非收藏,margin-left:auto) / `with-fav`(已收藏,margin-left:0)；常驻星标 `.entry-fav-persistent` 仍 `margin-left:auto` 在最右。两种情况下星标都在最右 → 位置一致。
需求2：`decoder` 与 `json` 共用 `code` 图标。改为 `decoder→'type'`、`json→'code'`。同步修改 `toast.tsx` 的 ACTION_META。
`tsc --noEmit` 通过。

## 修复二维码生成器尺寸拖放振荡 + 超出边框
根因：`.gen-preview` 的 `max-height:280px` 固定值，大尺寸/小窗口时图片溢出预览卡片。更深的根因：`<img>` 无 width/height 属性，每次生成新图浏览器解码瞬间塌缩到 0 再弹回，造成"先大再小再大"闪烁；且 max-height:280 + max-width:100% 对正方形 QR 各轴独立裁剪导致变形。后端 generateQr 异步无过期丢弃，慢速拖动多次防抖请求乱序返回也会让尺寸跳变。
修复：
- `src/features/qr/index.tsx`：`<img>` 加 `width={size} height={size}` 防解码塌缩；新增 `seq` 序号仅在最新响应生效；表单区加 `gen-form-section` 类并压缩 textarea 到 2 行。
- `src/styles/viewers.css`：`.gen-preview` 改 `max-height:100%` + `object-fit:contain` 等比缩放；`.gen-preview-section` 加 `overflow:hidden` + `flex:1 1 auto` 占满剩余空间；新增 `.gen-form-section { flex:0 0 auto }`。
- `src-tauri/src/command/viewer.rs`：qr-viewer 窗口 520×620 → 600×760（显示更大二维码）。
显示尺寸 = min(size, 容器)，窗口可拖大逼近 640。
