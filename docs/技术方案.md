## 1. 技术栈

### 1.1 框架选型

| 层 | 选型 | 理由 |
|---|------|------|
| GUI 框架 | `egui` 0.35 | 即时模式，原生渲染，无 WebView2 开销 |
| 窗口管理 | `winit` 0.30 + `egui-winit` 0.35 | 替代 eframe，掌控窗口显隐和焦点行为 |
| 渲染后端 | `egui-wgpu` 0.35 | wgpu 后端（D3D12/Vulkan/Metal），跨平台，`features = ["winit"]` |
| wgpu 阻塞 | `pollster` 0.4 | 同步 async 初始化（`block_on`） |
| 剪贴板 | `clipboard-rs` 0.3.5 | 文本/图片/文件读写 + `ClipboardWatcherContext` 监听 |
| 数据库 | `rusqlite` 0.40 | features = `["bundled"]` |
| 系统托盘 | `tray-icon` 0.24 | tauri 团队维护，后台线程消息泵 |
| 全局热键 | `global_hotkey` 0.8 | tauri 团队维护，try_recv 轮询 |
| 键盘钩子 | 自写 WH_KEYBOARD_LL | 直接使用 `windows` crate 的 `SetWindowsHookExW` |
| Win32 API | `windows` 0.62 | 来源追踪/开机自启/单实例/GDI/OpenGL |
| 图片编解码 | `image` 0.25 | features = `["png", "bmp", "jpeg"]` |
| 序列化 | `serde` 1 + `serde_json` 1 | settings.json |
| 哈希 | `sha2` 0.10 | 内容去重 |
| 日志 | `log` 0.4 + `env_logger` 0.11 | 文件日志 |
| UUID | `uuid` 1 | 图片文件命名 |

### 1.2 不使用 eframe 的原因

| 问题 | 说明 |
|------|------|
| `eframe::ViewportCommand::Visible(false)` | 导致事件循环停滞，无法重新显示窗口 |
| 我们的需求 | 失焦 → `window.set_visible(false)` 完全隐藏；唤出 → `set_visible(true)` |
| 解决方案 | `egui-winit` + `egui-wgpu` + `winit` 自行管理事件循环 |
| 样板代码 | ~200 行，一次编写后续不改 |
| UI 代码 | 与 eframe 完全相同（`ctx.run()` 内仍是 egui 代码） |

## 3. 核心模块设计

### 3.1 Clipboard 管道（ClipboardService）

由 `clipboard/service.rs` 中的 `ClipboardService` 管理，App 只需委托：

```text
系统剪贴板变化
     │
     ▼
clipboard-rs ClipboardWatcherContext
     │  on_clipboard_change() 回调（后台线程）
     │
     ▼
capture_current() → CapturedData
     │  1. 格式检测: has(Text/Image/Files)
     │  2. 读取: get_text() / get_image() / get_files()
     │  3. 哈希去重: SHA256
     │  4. TagMask 计算: Text/Image/URL/File
     │
     ▼
crossbeam channel 发送到主线程
     │
     ▼
ClipboardService::handle(data, repo, image_store)
     │  1. SelfWriteTracker 自写检查 (5s TTL) → skip
     │  2. FiloStack.auto_exit_if_needed (图片/文件退出队列)
     │  3. get_clipboard_source() → exe + title
     │  4. repo.upsert_entry(content_hash 去重)
     │  5. repo.upsert_format (CF_UNICODETEXT / CF_DIB / CF_HDROP)
     │  6. filo_stack.push(text)（仅 queue 模式）
     │
     ▼
App.refresh_entries() (UI 刷新)
```

`ClipboardService` 同时持有 `FiloStackService` 和 `SelfWriteTracker`，
形成清晰的 seam：可构造  → `handle(data, &repo, &image_store)` → 验证 DB。

### 3.2 复制操作流（用户点击复制按钮 / 按 Enter / Ctrl+C / Ctrl+1~9）

```text
用户触发复制
     │
     ▼
1. 根据条目类型读取内容
     │  - 文本: clipboard_format.content (CF_UNICODETEXT)
     │  - 图片: 读取 image_store .png → clipboard-rs set_image()
     │  - 文件: clipboard-rs set_files()
     │
     ▼
2. SelfWriteTracker.mark(text_hash)  ← 避免重新捕获
     │
     ▼
3. clipboard-rs set_text/set_image/set_files()
     │
     ▼
4. 隐藏窗口: window.set_visible(false)
```

### 3.3 FiloStack 粘贴顺序控制

```
【正常模式】  ← 默认
  不安装 Hook，用户按 Ctrl+V 走系统原始剪贴板

【队列模式】  ← 用户手动切换
  安装 WH_KEYBOARD_LL 钩子
  钩子线程运行 GetMessage 消息泵
  拦截 Ctrl+V:
    1. 检查 selfPaste guard（500ms TTL）→ 放行
    2. Pop() 从队列 FIFO 取出一条
    3. SelfWriteTracker.mark()
    4. clipboard-rs set_text()
    5. CallNextHookEx 放行（此时剪贴板已替换）
  自动退出条件：
    - 用户复制了非文本内容（图片/文件）→ 自动切回 normal
    - 队列清空 → 保持 queue 模式，继续积累新条目
```

## 4. 数据库设计

### 4.1 表结构

```sql
CREATE TABLE IF NOT EXISTS clipboard_entry (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    content_hash    TEXT NOT NULL UNIQUE,       -- SHA256 哈希（去重键）
    source_exe      TEXT NOT NULL DEFAULT '',   -- 来源进程路径
    source_title    TEXT NOT NULL DEFAULT '',   -- 来源窗口标题
    tag_mask        INTEGER NOT NULL DEFAULT 0, -- 标签位掩码
    is_favorite     INTEGER NOT NULL DEFAULT 0, -- 是否收藏
    content_length  INTEGER NOT NULL DEFAULT 0, -- 文本长度（排序用）
    created_at      TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%f', 'now')),
    updated_at      TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%f', 'now'))
);
CREATE INDEX IF NOT EXISTS idx_entry_updated_at ON clipboard_entry(updated_at DESC, id DESC);

CREATE TABLE IF NOT EXISTS clipboard_format (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    entry_id    INTEGER NOT NULL REFERENCES clipboard_entry(id) ON DELETE CASCADE,
    format_type INTEGER NOT NULL,              -- CF_UNICODETEXT(13)/CF_DIB(8)/CF_HDROP(15)
    content     TEXT,                          -- 文本内容
    file_path   TEXT,                          -- 图片相对路径 (images/YYYY-MM-DD/uuid.png)
    format_hash TEXT NOT NULL,                 -- 单格式 SHA256
    UNIQUE(entry_id, format_type)
);
CREATE INDEX IF NOT EXISTS idx_format_entry ON clipboard_format(entry_id);

PRAGMA foreign_keys = ON;
PRAGMA journal_mode = WAL;
PRAGMA busy_timeout = 5000;
```

### 4.2 表结构与旧版的差异

| 差异点 | 旧版 | 新版 |
|--------|------|------|
| 数据目录 | `%APPDATA%/jPaste/` | `%APPDATA%/jPastev2/` |
| 图片格式 | `.png` + `.dib` 双文件 | 仅 `.png`（clipboard-rs 原生支持） |
| 后向迁移 | 有（is_favorite, content_length）| 无（DLL 直接用最终版） |

### 4.3 关键查询

```sql
-- 光标分页查询（自动加载）
SELECT e.*, f.format_type, f.content, f.file_path
FROM clipboard_entry e
LEFT JOIN clipboard_format f ON f.entry_id = e.id
WHERE (tag_mask & ?) != 0          -- 标签过滤，0=全部
  AND (updated_at < ? OR (updated_at = ? AND e.id < ?))  -- 光标
  AND (content LIKE ?)             -- 全文搜索（可选）
ORDER BY updated_at DESC, e.id DESC
LIMIT 21;                          -- 20条+1条检测 has_more

-- 统计
SELECT COUNT(*), COALESCE(SUM(content_length), 0) FROM clipboard_entry;

-- 清理过期货
DELETE FROM clipboard_entry
WHERE updated_at < datetime('now', '-' || ? || ' days')
  AND is_favorite = 0;
```

### 5.5 Toast 通知（纯 Win32 分层窗口）

在后台线程运行独立 Win32 消息泵，使用 `windows` crate 创建 WS_EX_LAYERED 窗口：

1. `RegisterClassW` 注册窗口类 → `CreateWindowExW(WS_EX_TOOLWINDOW|WS_EX_TOPMOST|WS_EX_LAYERED)`
2. `SetLayeredWindowAttributes(flags=LWA_ALPHA, alpha=opacity)` 控制透明
3. 后台线程 `GetMessageW` 循环 + `crossbeam_channel` 接收命令
4. 显示 → `SetWindowPos(HWND_TOPMOST, 右下角)` + `SetTimer(3s)`
5. 自动隐藏 → `WM_TIMER` → `ShowWindow(SW_HIDE)` + `KillTimer`
6. GDI 绘制：`FillRect` 深色背景 + 紫色左侧强调条

## 6. 事件循环架构

```text
┌─────────────────────────────────────────────────────────────────┐
│                       主线程 (winit 事件循环)                      │
│                                                                   │
│  event_loop.run(move |event, target| {                            │
│      target.set_control_flow(ControlFlow::Poll);                  │
│                                                                   │
│      match event {                                                │
│          WindowEvent::Focused(false) => {                         │
│              if !pinned { window.set_visible(false) }             │
│          }                                                        │
│          WindowEvent::CloseRequested => {                         │
│              window.set_visible(false)  // 隐藏而非退出            │
│          }                                                        │
│          WindowEvent::RedrawRequested => {                        │
│              process_clipboard_events()                           │
│              poll_global_events()                                 │
│              input = egui_winit_state.take_egui_input(&window)    │
│              output = egui_ctx.run_ui(input, |ctx| ui_update())   │
│              egui_winit_state.handle_platform_output(…)           │
│              primitives = egui_ctx.tessellate(shapes, pp)         │
│              painter.paint_and_update_textures(…)                 │
│          }                                                        │
│      }                                                            │
│      AboutToWait => window.request_redraw()                       │
│  });                                                              │
│                                                                   │
├─────────────────────────────────────────────────────────────────┤
│                    后台线程 (Clipboard Watcher)                    │
│  clipboard-rs ClipboardWatcherContext                             │
│      on_clipboard_change() → capture_current()                    │
│      → Sender<ClipboardEvent> 发送到主线程                         │
├─────────────────────────────────────────────────────────────────┤
│                    后台线程 (FiloStack Hook)                      │
│  自写 WH_KEYBOARD_LL 钩子消息泵                                   │
│      → 检测 Ctrl+V → Pop → clipboard-rs set_text                 │
├─────────────────────────────────────────────────────────────────┤
│                    后台线程 (Toast)                                │
│  Win32 分层窗口 (WS_EX_LAYERED + WS_EX_TOPMOST)                   │
│  GDI 绘制 + SetTimer 自动隐藏                                     │
├─────────────────────────────────────────────────────────────────┤
│                    后台线程 (System Tray)                          │
│  tray-icon 消息泵 → try_recv 轮询                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 12.2 测试基础设施

- 内存 SQLite（`rusqlite::Connection::open_in_memory`）— 无需真实数据库文件
- `tempfile::tempdir` — 隔离的临时目录用于设置和图片文件测试
- 共享 `tests/common/mod.rs` — `temp_repo()` / `temp_settings()` 工具函数

### 12.3 架构洞察：seam 模式

架构深化（**candidate 1 & 2**）创建了清晰的可测试 seam：

```
ClipboardService::handle(data, &repo, &image_store) → Result<Option<i64>>
    ↑ 无需 GUI、无需系统剪贴板、纯业务逻辑

ops::copy_entry(repo, tracker, id) → Result<Option<String>>
ops::delete_entry(repo, entries, index, sel) → Result<()>
    ↑ entry CRUD 不依赖 App 结构体
```

这些 seam 是所有集成测试的基础：构造 → 调用 → 断言，不启动 winit/egui。
