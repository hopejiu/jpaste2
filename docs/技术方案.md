# jPaste v2 技术方案

> 从 jPaste (Go + Wails3 + React + JS) 重构为 Rust + Tauri 2 + Preact + TypeScript
>
> 最后更新：2026-07-02

## 1. 概述

### 1.1 项目目标

将现有 jPaste（Windows 剪贴板管理器）用 Rust + Tauri 2 重构，同时：
- **简化存储** — 单表 SQLite 替代双表，仅存储纯文本和图片
- **同语言化** — 全栈 Rust + TypeScript 单一构建链路
- **保持功能** — 全部功能（队列粘贴、查看器、Action 模块）不丢失
- **裁减冗余** — 放弃来源应用追踪、放弃多格式存储（HTML/RTF/HDROP）

### 1.2 技术栈总览

| 层 | 技术 | 备注 |
|----|------|------|
| 桌面框架 | Tauri 2 | Rust 后端 + WebView 前端 |
| 后端语言 | Rust | 2021 edition |
| 前端框架 | Preact 10 | 轻量 React 替代 |
| 前端语言 | TypeScript 6 | Vite 8 bundler |
| 前端路由 | wouter | 轻量路由 (<2KB) |
| 前端图标 | lucide-preact | 与 v1 一致 |
| 前端状态 | Preact Signals | 无额外依赖 |
| 剪贴板 | clipboard-rs 0.3.5 | 读写 + 监听 + 文件列表检测 |
| 全局热键 | tauri-plugin-global-shortcut | Tauri 官方插件 |
| 键盘钩子 | windows crate (Win32 FFI) | WH_KEYBOARD_LL + keybd_event |
| 数据库 | rusqlite + SQLite | 单表 entries |
| 图片存储 | 文件系统 | `{appdata}/jpaste2/images/{YYYY-MM-DD}/{uuid}.png` |
| 序列化 | serde / serde_json | settings.json |
| 日志 | tauri-plugin-log / env_logger | 统一日志 |

### 2.2 模块职责

#### `lib.rs`（应用入口）
- 初始化 AppData 目录
- 创建并注册所有 Tauri 命令
- 初始化 Repository（SQLite 连接）
- 创建各个 Service 实例并注入依赖
- 注册 Tauri 插件（global-shortcut, opener）
- 启动 clipboard 监听（clipboard-rs watcher）
- 注册系统托盘
- 配置单实例限制

#### `clipboard/`
**基于 clipboard-rs 的封装层**，而非直接 Win32 API。

核心功能：
- **监听** — 实现 `ClipboardHandler` trait，通过 `ClipboardWatcherContext` 接收变更通知
- **读取** — 变更时读取当前剪贴板：
  - `get_text()` — 纯文本 → 存储到 `content` 列
  - `get_image()` — 图片 → 保存到图片文件，`image_path` 存路径
  - `available_formats()` — 检测文件列表 → 用于打 `file` 标签
- **写入** — `set_text()` 写回剪贴板（供粘贴队列使用）
- **粘贴模拟** — `keybd_event` 模拟 Ctrl+V（通过 windows crate 的 `SendInput`）

> 注意：`clipboard-rs` 在 Windows 上使用 `CF_DIBV5`（优先）/ `CF_PNG` 格式存储图片。

#### `hook/`（新增模块）
WH_KEYBOARD_LL 全局键盘钩子，用于拦截 Ctrl+V 实现队列粘贴消费。

```
Service.SetMode("queue")
  └→ hook.startHook()
       └→ SetWindowsHookEx(WH_KEYBOARD_LL, callback, ...)
            └→ 检测 KBDLLHOOKSTRUCT vkCode == VK_V && Ctrl 按下
                 └→ filoStack.Pop() → clipboard.writeText() → keybd_event Ctrl+V
```

**关键设计**：
- 钩子在独立的消息循环线程中运行（需要 `GetMessage` 循环）
- `SelfWriteTracker` 防止自写入的回环
- `selfPasteUntil` 时间戳标记防止自粘贴检测
- 粘贴队列为空时放行原生 Ctrl+V

#### `service/history.rs`
历史记录 CRUD，通过 `repository/` 访问 SQLite。

Tauri 命令暴露列表（每个命令加 `#[tauri::command]`）：

| 命令 | 参数 | 返回 | 说明 |
|------|------|------|------|
| `get_entries` | search, tagMask, cursor, limit, sortField, sortOrder | `Vec<Entry>` | 游标分页查询 |
| `delete_entry` | id | `()` | 删除条目 + 图片文件 |
| `toggle_favorite` | id, value | `()` | 切换收藏 |
| `get_entry_content` | id | `String` | 获取完整文本内容 |
| `get_entry_image` | id | `String` (base64) | 获取图片 base64 |
| `clear_all` | keepFavorites | `()` | 清空全部 |
| `cleanup` | retainDays | `(deleted, imagePaths)` | 过期清理 |

#### `service/settings.rs`
`settings.json` 的读写管理。与 v1 相同字段 + 变更回调模式。

**Settings.Data 结构**（带 serde 序列化）：
```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Data {
    pub hotkey: String,            // "Alt+V"
    pub retain_days: u32,          // 30
    pub default_action: String,    // "copy" / "paste"
    pub auto_start: bool,
    pub start_minimized: bool,
    pub notify_enabled: bool,
    pub paste_order: String,       // "normal" / "queue"
    pub action_config: Value,      // 透传 JSON（serde_json::Value）
    pub sort_field: String,        // "updated_at" / "content_length"
    pub sort_order: String,        // "asc" / "desc"
    pub theme: String,             // "a" / "b" / "c"
    pub auto_clear_search: bool,
    pub auto_clear_seconds: u32,
    pub auto_hide_after_copy: bool,
    pub notify_opacity: u8,        // 0-100
}
```

Observer 模式：设置变更时通知 `history`（重新执行 cleanup）、`filostack`（切换模式）、全局热键（重新注册）。

Tauri 命令：
- `get_settings` → `Data`
- `save_settings(data)` → `Result`

#### `service/filostack.rs`
粘贴队列服务。使用 `container/linked-list`（Rust 标准库 `LinkedList`）存储队列项。

**Strategy 模式**（与 v1 一致）：
```rust
trait PasteStrategy {
    fn pop(&self, list: &mut LinkedList<String>) -> Option<String>;
    fn mode_name(&self) -> &str;
}

struct QueueStrategy;  // FIFO: pop_front()
```

FiloStack 在队列模式下：
1. `Push(text)` — clipboard 监听回调中调用，将新内容推入队列
2. `Pop() → Option<String>` — 键盘钩子回调中调用，取出下一个待粘贴内容
3. `writeText(text)` — 写入剪贴板 → `keybd_event` 模拟 Ctrl+V

#### `service/toast.rs`
Toast 通知服务。通过 Tauri 多窗口 API 创建轻量物理窗口。

流程：
```
收到 clipboard 变化 → notifyEnabled? → 创建 Toast 窗口
  → 加载 toast.html?title=xxx&message=xxx → 显示 3 秒 → 自动关闭并销毁窗口
```

**与 v1 差异**：v1 保持窗口常开（offscreen），v2 每次创建→显示→销毁。更简洁，但需要处理窗口创建延迟（Tauri 多窗口创建约 10-50ms）。

#### `service/fileop.rs`
文件操作：
- `open_in_explorer(path)` — 在资源管理器中打开/选中
- `preview_text(id)` — 创建临时文件并在编辑器中打开

通过 `tauri-plugin-opener` 实现资源管理器操作。

#### `viewer/`
4 个查看器窗口，每个包含：
- Tauri 端：创建窗口的 Rust 方法，通过 `app.get_webview_window()` API 创建新窗口
- 前端端：独立的 Preact 页面渲染（`/viewer/json?id={id}` 等路由）

创建方式：
```rust
#[tauri::command]

    let builder = tauri::WebviewWindowBuilder::new(
        &app,

        tauri::WebviewUrl::App(format!("/viewer/json?id={}", id).into()),

    builder
        .title("JSON 查看")
        .inner_size(1200.0, 800.0)
        .min_inner_size(600.0, 400.0)
        .build()?;

```

#### `action/`
内容识别操作模块。在 Rust 端运行**检测逻辑**，前端负责**渲染 UI**。


检测结果通过 `get_actions(content)` 命令返回给前端，前端按 priority 展示前 3 个匹配的操作按钮。

#### `util/`
- **hashutil** — SHA256 哈希（`sha2` crate）
- **textutil** — 文本截断
- **dibutil** — DIB→BMP 头部拼接
- **tracker** — `SelfWriteTracker`（自写入跟踪，用于防回环）

## 3. 数据库设计

### 3.1 单表存储（简化版）

```sql
CREATE TABLE IF NOT EXISTS entries (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    content_hash   TEXT NOT NULL UNIQUE,
    content        TEXT NOT NULL DEFAULT '',        -- CF_UNICODETEXT 内容
    image_path     TEXT NOT NULL DEFAULT '',        -- 图片文件路径（相对 app data）
    tag_mask       INTEGER NOT NULL DEFAULT 0,
    is_favorite    INTEGER NOT NULL DEFAULT 0,
    content_length INTEGER NOT NULL DEFAULT 0,     -- content 的字节长度
    created_at     TEXT NOT NULL,                    -- ISO 8601 毫秒精度
    updated_at     TEXT NOT NULL
);

CREATE INDEX idx_entries_updated ON entries(updated_at DESC);
CREATE INDEX idx_entries_hash ON entries(content_hash);
CREATE INDEX idx_entries_favorite ON entries(is_favorite);
```

### 3.2 与 v1 对比

| 项目 | v1 | v2 |
|------|----|----|
| 表数 | 2（clipboard_entry + clipboard_format） | 1（entries） |
| 多格式 | 每个格式一行（text/html/rtf/dib/hdrop） | 弃用 |
| 图片路径 | `clipboard_format.file_path` | `entries.image_path` 直属列 |
| 来源追踪 | `source_exe` + `source_title` | 弃用 |
| 搜索 | JOIN clipboard_format | 直接在 `content` 列 LIKE |
| 游标分页 | `(updated_at, id)` 复合游标 | 同左 |
| 排序 | `updated_at` / `content_length` | 同左 |

### 3.3 标签位掩码（与 v1 一致）

| Tag | Bit | 判断依据 |
|-----|-----|----------|
| `text` | 1 (1<<0) | 有纯文本，无图片 |
| `image` | 4 (1<<2) | 图片路径非空 |
| `url` | 8 (1<<3) | content 以 http:// 或 https:// 开头 |
| `file` | 16 (1<<4) | content 符合 Windows 路径格式 |
| `favorite` | 32 (1<<5) | is_favorite=1（虚拟标签，前端过滤用） |

### 3.4 游标分页

与 v1 一致：复合游标 `(updated_at, id)`，每页 20 条。

```sql
SELECT id, content_hash, content, image_path, tag_mask,
       is_favorite, content_length, created_at, updated_at
FROM entries
WHERE (updated_at < ? OR (updated_at = ? AND id < ?))
  AND tag_mask & ? != 0    -- 可选标签过滤
  AND content LIKE ?        -- 可选搜索
ORDER BY updated_at DESC, id DESC
LIMIT 21;                   -- 取 21 条来判断 hasMore
```

## 6. 剪贴板管道

### 6.1 数据流

```
┌──────────┐    clipboard-rs 监听     ┌──────────────┐
│ 系统剪贴板 │ ──── AddClipboardFormatListener ──▶│ clipboard/    │
│          │                         │ handler.rs   │

                                            │ on_clipboard_change()
                                            ▼

                              │  get_text() / get_image()  │
                              │  available_formats()       │
                              └───────────┬───────────────┘
                                          │
                              ┌──────────▼───────────┐
                              │  去重判断              │
                              │  SHA256(content)       │
                              │  DB entries 已存在?   │
                              └──────┬──────────┬─────┘
                            是/更新  │          │ 否/插入

                              │ 更新    │  │ 新建 Entry  │
                              │updated_at│  │ 持久化格式   │
                              └─────┬───┘  └─────┬──────┘
                                    │              │
                                    ▼              ▼
                              ┌──────────────────────────┐
                              │ 发送事件到前端             │
                              │ clipboard-updated          │
                              │ filostack.Push(text)       │
                              │ Toast（如启用）             │
                              └──────────────────────────┘
```

### 6.2 检测逻辑

```rust
// clipboard/handler.rs
impl ClipboardHandler for ClipboardManager {
    fn on_clipboard_change(&mut self) {
        // 1. 防自写入检测
        if self.self_tracker.is_self_write() { return; }

        // 2. 读取内容
        let text = self.ctx.get_text().unwrap_or_default();
        let image = self.ctx.get_image().ok();
        let formats = self.ctx.available_formats().unwrap_or_default();

        // 3. 计算标签
        let tag_mask = compute_tag_mask(&text, image.is_some(), &formats);

        // 4. 确定主内容用于哈希和去重
        let primary = if !text.is_empty() { text.clone() }
                      else { /* image bytes hash */ };
        let hash = sha256_hex(&primary);

        // 5. 去重插入
        if let Ok(true) = self.repo.upsert_dedup(&hash, tag_mask, &text) {
            // 已存在→更新时间戳，跳过后续
            return;
        }
        let entry_id = self.repo.insert_entry(&hash, &text, tag_mask, text.len())?;

        // 6. 图片处理
        if let Some(img) = image {
            let path = self.image_store.save(img, entry_id)?;
            self.repo.set_image_path(entry_id, &path)?;
        }

        // 7. 通知依赖方
        self.app.emit("clipboard-updated", payload);
        self.filostack.push(&text);
        if self.settings.notify_enabled {
            self.show_toast(&text);
        }
    }
}
```

## 7. 队列粘贴模式（FiloStack）

### 7.1 状态转换

```
                   ┌──────────┐
       SetMode     │  normal   │  SetMode
     "queue" ──▶  │           │◀── "normal"
                   └─────┬────┘
                         │ startHook()
                         ▼
              ┌──────────────────┐
              │     queue        │
              │  Hook 激活       │
              │  LinkedList 非空  │
              └──────┬──┬───────┘
                Push │  │ Pop
         clipboard  │  │ WH_KEYBOARD_LL
         变更回调    ▼  ▼ 检测 Ctrl+V
```

### 7.2 自写入保护

```rust
pub struct SelfWriteTracker {
    hash: String,
    expires_at: Instant,
}

impl SelfWriteTracker {
    pub fn mark(text: &str);         // 写入前记录
    pub fn is_expired(&self) -> bool;  // 500ms 超时
    pub fn is_self_write(&self) -> bool; // 当前剪贴板内容匹配
}
```

流程：

```
filostack 从队列弹出内容
  → SelfWriteTracker.mark(text)
  → clipboard.set_text(text)  ← 触发 WM_CLIPBOARDUPDATE
  → on_clipboard_change() 检测 is_self_write() → 跳过处理
  → keybd_event 模拟 Ctrl+V（实际粘贴到当前焦点窗口）
```

## 8. 窗口管理

### 8.1 窗口清单

| 窗口 | 类型 | 创建时机 | 行为 |
|------|------|----------|------|
| **主窗口** | 应用主窗口 | 启动时 | 隐藏到托盘 / 显示 |
| **Toast** | 物理窗口 | 每次通知创建 | 右下角无框显示 3s → 自动关闭销毁 |
| **JSON 查看器** | 物理窗口 | 用户点击 action | 独立窗口，jsoneditor tree/code 模式 |
| **图片查看器** | 物理窗口 | 双击图片条目 | 独立窗口，自适应缩放，← → 切换 |
| **Curl 调试器** | 物理窗口 | 检测到 curl 命令 | 独立窗口，HTTP 调试（解析/编辑/发送） |
| **WS 调试器** | 物理窗口 | 检测到 ws:// 地址 | 独立窗口，WebSocket 收发 |

### 8.2 Toast 窗口生命周期

```
Step 1: clipboard 变更 → notify enabled
Step 2: WebviewWindowBuilder::new("toast", URL)
         → URL 参数携带 title, message, opacity
Step 3: window.show() → 计算屏幕右下角位置
Step 4: window.set_position(x, y)
Step 5: 3 秒后 → window.close() → 窗口销毁
```

### 8.3 主窗口显隐

| 事件 | 行为 |
|------|------|
| 关闭按钮点击 | hide（隐藏到托盘，不退出） |
| 失去焦点 | auto-hide（除非 pinned） |
| Alt+V 全局热键 | toggle 显示/隐藏 |
| 托盘左键点击 | toggle 显示/隐藏 |
| 托盘右键菜单 | 显示 / 设置 / 退出 |

## 9. 事件系统

Tauri 2 的 `app.emit()` / `window.listen()` 替代 Wails Events：

| 事件名 | 方向 | 含义 |
|--------|------|------|
| `clipboard-updated` | Rust → Preact | 新剪贴板条目已保存 |
| `window-shown` | Rust → Preact | 窗口已显示（触发焦点搜索框等） |
| `window-hiding` | Rust → Preact | 窗口即将隐藏（清理状态） |
| `paste-order-changed` | Rust → Preact | 粘贴模式切换 |
| `navigate` | Rust → Preact | 系统托盘点击"设置"跳转 |
| `frontend-log` | Preact → Rust | 前端日志回传 |
| `toast-show-preview` | Preact → Rust | 设置页预览 Toast |
| `toast-hide-preview` | Preact → Rust | 关闭预览 Toast |

### Rust 端发送事件

```rust
// lib.rs
app.emit("clipboard-updated", serde_json::json!({
    "id": entry_id,
    "content_preview": &text[..40.min(text.len())],
}))?;
```

### Preact 端接收事件

```typescript
// hooks/use-events.ts
import { listen } from '@tauri-apps/api/event';
import { useEffect } from 'preact/hooks';

export function useClipboardEvent(handler: (payload: any) => void) {
  useEffect(() => {
    const unlisten = listen('clipboard-updated', (e) => handler(e.payload));
    return () => { unlisten.then(fn => fn()); };
  }, []);
}
```

## 10. 内容识别操作模块（Action Module）

### 10.1 前端模块定义

与 v1 一致：每个模块一个文件，导出 `detect()` 方法和可选的 `Component`。

```typescript
// actions/registry.ts
export interface ActionModule {
  id: string;
  label: string;
  icon: string;          // lucide-preact 图标名
  priority: number;
  detect(content: string): boolean;
  handler?(content: string, entryId: number): void;
  Component?: (props: { content: string; entryId: number; onClose: () => void }) => JSX.Element;
}
```

### 10.2 模块清单

| 模块 | 检测规则 | 行为 |
|------|----------|------|
| `math` | 纯数字/运算符/括号，至少一个运算符 | 模态框表达式求值 |
| `json` | 以 `{` 或 `[` 开头，合法 JSON | 创建独立窗口打开 jsoneditor |
| `url` | 以 `http://` 或 `https://` 开头 | `tauri-plugin-opener` 打开浏览器 |
| `folder` | 以 `X:\` 或 `\\` 开头的 Windows 路径 | 资源管理器打开路径 |
| `base64` | Base64 字符集，长度>4，mod4=0 | 模态框解码（输入→解码→复制） |
| `unicode` | 包含 `\uXXXX` 模式 | 模态框转义解码 |
| `curl` | 以 `curl ` 开头（不区分大小写） | 创建独立窗口，curlconverter 解析 |
| `ws` | 以 `ws://` 或 `wss://` 开头 | 创建独立窗口，WebSocket 收发 |
| `timestamp` | 纯数字 10 位或 13 位，范围 2000-2100 | 模态框时间戳↔日期双向转换 |

### 10.3 检测流程

```
条目内容变动 → frontend 调用 Rust 端 get_actions(entryId)
  → Rust 端遍历注册的 ActionModule
  → 返回匹配的模块列表（按 priority 排序，最多 3 个）
  → 在 EntryItem 上显示操作按钮
```

## 12. 数据迁移（从 v1）

v1 的 `clipboard.db` 结构与 v2 完全不同，**不做自动迁移**。v2 启动全新数据库。

但可以提供一个手动导入命令：
```
#[tauri::command]
async fn import_from_v1(v1_db_path: String) -> Result<ImportResult>
```
读取 v1 的 `clipboard_entry` + `clipboard_format` 表，合并多格式为单条记录（优先取 CF_UNICODETEXT，其次图片），插入 v2 的 entries 表。
